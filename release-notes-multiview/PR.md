Add per-layout EPG forwarding and fix plugin load failure (v0.2.2)

Each multiview layout now has an EPG source mode setting. The default ("Placeholder") keeps the existing built-in programme entry behaviour. Selecting "Forward from channel" enables a channel picker; the layout's XMLTV output then uses real programme data (title, subtitle, description) from the selected channel's EPG, falling back to placeholder if no data is available. Also fixes an intermittent plugin load failure during Dispatcharr reload cycles, where relative importlib calls would raise KeyError or ModuleNotFoundError if the parent package was not in sys.modules.

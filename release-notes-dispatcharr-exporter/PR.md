Add timeshift, channel, and plugin metrics; fix DB connection leak

Dispatcharr shipped a full catch-up/timeshift feature since our last metrics update, and this exporter had no visibility into it. This PR folds timeshift sessions and clients into the existing `dispatcharr_stream_*` and `dispatcharr_client_*` metric families (same treatment as live and VOD), adds EPG program info anchored to the catch-up position being watched, and reports the active M3U provider/profile for catch-up streams just like live. It also adds new opt-in metrics for per-channel info (group, source count, catch-up config) and installed plugin health. Along the way it fixes a DB connection leak in the metrics server: because the server runs outside Django's normal request cycle, connections opened during a scrape were never being closed, which under load could exhaust the database's connection limit and hang the whole app.
